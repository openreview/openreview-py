# Action Editors, Including OSS AEs

## Your Work

- Manage assigned papers from review start through decision and camera-ready verification.
- Assign and manage reviewers.
- Monitor review progress and reviewer-specific due dates.
- Submit decisions and rate reviewers.
- Check resubmission history, author-list differences, and reviewer continuity for papers you handle.
- OSS AE is an assignment marker on the same Action Editor workflow.

## Where To Go

Use the Action Editor Console for pending tasks, assigned papers, assignment availability, and expertise. Open papers from that console to keep the AE role context.

## Main Actions

| Action | When to use it |
|---|---|
| Assign Reviewers | Open the paper-specific reviewer assignment page and add enough eligible reviewers for an assigned paper. |
| Reviewer assignment controls | Adjust reviewer assignments and set the due date for newly assigned reviewers from the assignment page while the paper state allows it. |
| Reviewer removal | Remove an assigned reviewer who has not submitted a review from the assignment page when reviewer management is still allowed. |
| Editorial Comment | Send paper communication to selected allowed readers. |
| Decision | Submit an editorial decision, including early rejection when appropriate. |
| Reviewer Rating | Optionally rate submitted reviews or report reviewer problems when rating actions are available. |
| Verify camera-ready revision | Approve final material or request camera-ready changes. |

## Notes

Use the normal Decision action for early rejection; there should not be a separate Desk Reject button on the AE paper page.

Reviewer recommendation is part of the review form, so there should not be a separate recommendation task.

On the Assign Reviewers page, set the review due date when assigning reviewers
through previous-reviewer, auto-assign, search, or external-invite flows.
Existing reviewer due dates are not edited from the paper forum.

Editorial comments should use the visible paper action and should only be
readable by the roles selected by the form and allowed by the paper state.

If you also hold another role, enter papers from the Action Editor Console when acting as AE. AE actions should use the assigned AE role context for that paper.

For resubmissions, check previous-submission links and author-name differences as editorial context; they should not create a separate approval task or pause reviewer assignment.

Resubmissions may show stable reviewer numbers such as `Reviewer 22tK (Reviewer 1)` so you can connect reviewer continuity across rounds even when OpenReview generates new current-round anonymous reviewer ids.
