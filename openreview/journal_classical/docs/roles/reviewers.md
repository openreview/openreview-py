# Reviewers

## Your Work

- Review assigned JMLR papers.
- Keep your assignment availability and maximum active review load current.
- Submit the review and recommendation together in the review form.
- Use paper-specific contact with the Action Editor when review-stage questions arise.

## Where To Go

Use the Reviewer Console for pending reviews, assigned papers, availability, and expertise. Open papers from that console to stay in reviewer role context.

## Main Actions

| Action | When to use it |
|---|---|
| Submit Review | Provide comments, recommendation, and optional review file. |
| Edit/Read Review | Reopen your submitted review when the paper state allows it. |
| Contact Action Editor | Ask the handling editor a restricted paper-specific question. |
| Assignment Availability | Pause or resume new reviewer assignments. |
| Max Active Reviews | Set your preferred active review load. |
| Top Reviewer Listing | Choose whether to be listed on the website if you qualify for Top Reviewer recognition. |

## Notes

The review form includes the recommendation; there is no separate recommendation task.

Before release, authors and other reviewers should not see your review text or recommendation. After release, review content is still anonymous to authors and peer reviewers.

If you also hold another JMLR role, enter papers from the Reviewer Console when acting as reviewer. The reviewer page should not offer AE, EIC, production-editor, or author signatures.

Top Reviewer listing is website recognition only. Choosing the opt-out setting does not affect review assignments, availability, load, cooldown, conflicts, or reviewer permissions.

Reviewer recruitment and volunteer responses are described in
[Role Recruitment And Volunteer Responses](../workflow/role-recruitment.md).
